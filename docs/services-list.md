**MedInsight AI Cloud Stack (Current):**

* **Loveable:** The AI-first integrated development environment (IDE) is used to rapidly build, iterate on, and generate the initial React frontend and user interface (UI) components.   
* **Antigravity:** The technical collaborator that helps us bridge the gap between a vision or idea and a real, functional feature or product.  
* **Supabase:** The backend-as-a-service that stores your data (chat history/user profiles), handles user authentication, and hosts the “Edge Functions” (the AI’s middleman).   
* **Google AI Studio:** The developer platform used to generate the **Gemini 2.5 Flash** API keys that provide the AI assistant’s medical intelligence.  
* **Google Cloud Platform (GCP):** The enterprise cloud environment where Marilyn’s billing account and project quotas are managed to ensure 1,500 free daily messages.  
* **GitHub:** The version control and code hosting platform that stores out “Source Code” and tracks every single change we’ve made to the bot.   
* **Vercel:** The deployment platform that hosts our frontend website, making it publicly accessible to users via a permanent URL.  
  * We are increasingly utilizing this over Loveable because Loveable’s free plan was preventing us from connecting to Supabase and would not allow us to override code. You must have a paid plan to edit any code on Loveable. 

**MedInsight AI Cloud Stack (Discontinued/Ceased Usage):**

* **Claude 3.5 Sonnet or GPT-4o:** When Loveable first generated our AI chat, these are the default project-level models that it would have defaulted to (not sure which one exactly). We have since transitioned away from Loveable to more technical tools.  
* **Gemini 2.0 Flash (Experimental):** We briefly used this older experimental model for initial troubleshooting, but have since fully migrated to the stable Gemini 2.5 Flash for response production.
